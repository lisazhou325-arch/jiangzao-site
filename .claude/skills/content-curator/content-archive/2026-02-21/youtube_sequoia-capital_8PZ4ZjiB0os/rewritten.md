## 终端正在吞噬 IDE

Zach Lloyd 在 Google 当了多年首席工程师,负责 Google Docs 的整个工程团队。协作软件的经验让他看到了另一个机会:开发者每天离不开的终端,体验糟糕得像上世纪的遗留系统。鼠标不能用、容易出错、学习曲线陡峭,但它在技术栈中的位置让它成为最强大的工具。

2020 年 Warp 拿到 2300 万美元融资时,没人预料到生成式 AI 会让这个决定变得如此正确。

"终端的形态天生适合代理工作",Zach 说,"所有交互都基于时间流,输入文本、输出文本、记录日志。你可以在终端里轻松管理多个代理的并发任务。" 今天 Warp 有 70 万活跃开发者,而产品已经从"现代化的终端"演变成"内置代理的工作台"。

更有意思的是竞争对手的动向。Cursor 在 2024 年 11 月推出的最新版本,主界面变成了对话框,文件树和编辑器退居次要位置——看起来越来越像 Warp。形态的融合正在发生:传统 IDE 像 Microsoft Word,终端像和电脑聊天,但当编码变成提示词驱动的工作流时,这两种工具都在朝同一个方向演化。

## 每五天增长 100 万美元,然后呢?

Warp 最初的定价策略很传统:订阅制,每月固定 AI 额度。Zach 的算盘是大部分用户用不完额度,平均下来能赚钱。但 2024 年下半年,开发者的使用量持续飙升,公司开始每次调用都亏钱。

"那段时间收入每五天增长 100 万美元",Zach 说,"但我们面临选择:可以继续烧钱、讲增长故事去融资,但利润率太差了。" 增速惊人,但每一美元背后都是更大的亏损。他们和投资人 Andreessen Horowitz 讨论后决定:不玩这个游戏。

新定价模型是 20 美元基础订阅加按量付费。用户抱怨声很大,增长确实放缓了,但这是唯一可持续的路径。"我们就是赢不了成本战",Zach 很直白,"Anthropic 在 2024 年 10 月推出 Claude Code,OpenAI 把 Codex 能力整合进 ChatGPT,Google 有 Gemini Code Assist。它们可以用模型利润补贴应用层。我们必须靠产品质量和整体体验赢得愿意多付 20、40、80 美元的专业开发者。"

这个决定背后是残酷的市场现实。根据 Gartner 2024 年的报告,AI 编码工具市场预计在 2025 年达到 18 亿美元,但企业采购渗透率仍不到 15%。模型提供商疯狂冲向应用层,因为他们看到了 API 业务的风险——当编码能力不再是前沿差异化优势时,很难在 token 上收取高溢价。

## 在 SWE-bench 上排第三,但这不是重点

Warp 在 2024 年 12 月的 SWE-bench Verified 上得分 38.2%,排名第三,仅次于 Anthropic 的 Claude 3.5 Sonnet(41.4%)和 OpenAI 的 o1-preview(40.6%)。在专门测试终端能力的 Terminal-bench 上,Warp 的得分是 72.8%,常年第一或第二。

但 Zach 认为真正的变革不在这些排行榜上,而在一个更根本的转变:代理不再需要开发者坐在键盘前启动。

"我们内部叫它们'守护代理'",Zach 说,"服务器崩溃、用户报告集中出现、有人提交安全漏洞——这些系统事件会直接触发代理在云端启动,开始工作。" 这不是科幻,Warp 团队已经在 Slack 和 Linear 里用上了。你在 Slack 里标记一个问题,代理会生成一个 300 行的 PR,基本不需要改。

这意味着工作台需要从个人工具变成编排平台。你需要看到团队所有代理的状态、日志、产出的 PR。代理会互相 review 代码,会往任务系统里提 ticket。Zach 说这是 Warp 2025 年最大的产品重点:"这会是变革性的。"

技术挑战也很实在:需要 agent SDK、云端托管服务、API 层来查询状态和接管控制、管理界面来追踪所有这些自主运行的东西。这让 Warp 从产品变成平台,也是为什么 Replit、Modal、Fly.io 等一批公司在做"代理托管"生意。

## 模型还在犯错,但已经不再写出无法编译的代码

Zach 每天用 Warp 改自己的代码库——一个复杂的大型 Rust 项目。他的评估很具体:"代理现在能处理中等复杂度的任务,如果你给足够指导的话。它们做不了整个大项目,我也不信任它们做根本性的架构决策。"

如果满分是 10 分,他给当前模型打 6 分。这是他基于日常使用的主观判断,但有一个客观指标支撑:"四五个月前,代理经常生成无法编译的代码。现在我这边基本 100% 能编译通过,这是个重要里程碑。" 

但代码里仍然有 bug,而且代理还不擅长从用户视角验证功能是否真的工作。这就是为什么 browser use 和 computer use 很重要——代理需要能够不仅写代码,还能像用户一样测试。Anthropic 在 2024 年 10 月发布了 Computer Use API 的 beta 版,OpenAI 和 Google 也在跟进。Zach 预计这很快会成为标配,尤其是当更多工作在远程环境完成时。

另一个瓶颈是上下文窗口。即使窗口变大了,让模型有效关注整个上下文仍然很难。而且每次都要从零开始,把一堆文件塞进去,没有持续学习能力。"它像个巨大的无状态机器",Zach 说,"你总是要先填满它才能放手让它干活。"

## 编码会被解决,然后呢?

"我确实认为编码会被模型解决",Zach 说,"我的意思是,限制因素会变成人类表达意图的能力——你想构建什么?怎么清晰地表达出来?" 

这是个有趣的倒退:开发者曾经直接用代码表达意图,现在要先用英语告诉模型,模型再生成代码。英语是模糊的,这重新引入了歧义。但这种方式效率高太多了。

Zach 认为几年内,你不需要最前沿的模型就能生成完美匹配人类意图的代码。"这就是为什么我不想做编码 token 的 API 生意——你很快就不需要在前沿,也无法收取高溢价。这也是为什么 Anthropic、OpenAI、Google 这么拼命冲向应用层,因为 API 层风险巨大。"

企业现在还把编码工具当生产力提升工具,不是劳动力替代。根据 Stack Overflow 2024 年的开发者调查,76% 的专业开发者在使用或计划使用 AI 编码工具,但只有 12% 的企业将其纳入正式的成本节约计划。很难量化效果,所以往往依赖工程师的主观感受或 DORA 指标这类间接数据。

但 Zach 预计这会改变:"当有公司在几乎没有工程师参与的情况下发布产品时,人们会开始计算有无工程师的成本差异。今天如果你说'这是个 20 万美元的代理,能替代 20 万美元的工程师',企业会说'什么?完全不行'。但这个认知会变。"

Warp 团队内部使用代理的方式差异很大。即使他们对写代码有严格标准,但对如何使用代理几乎没有共识。"没人被教过怎么用代理,甚至没有公认的最佳实践",Zach 说,"这个领域还太新了。"

凌晨三点,服务器崩溃。没有工程师被叫醒,一个守护代理自动启动,读取日志,定位问题,提交修复 PR,等待人类早上来 review。当编码本身不再是瓶颈,开发者的角色就变成了这个:在代理自主工作的间隙,做最后的判断和调整。Warp 要做的,就是那个让这一切发生的工作台。
